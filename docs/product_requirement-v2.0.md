一、产品概述

1.1 产品定位

职场英语助手是一个基于 AI 驱动的、面向外企职场人的个性化英语能力内化工具。

它不是翻译软件，也不是通用英语学习平台，而是帮助用户将日常工作中的真实英文沟通转化为可持续学习和复习的个人语料库，最终实现从“依赖 AI 润色”到“自主地道表达”的能力跃迁。

1.2 核心价值主张

> “把每一次 AI 帮你写的邮件，变成你下次能自己写出来的能力”

- 输入即学习：每次润色沟通内容的过程，同时也是积累个人语料的过程

- 场景化组织：自动按工作场景（邮件、会议、汇报、Jira 等）分类整理

- AI 驱动内化：通过 Flashcard、间隔复习、场景模拟等方式，帮助用户真正掌握表达

二、目标用户

2.1 核心用户画像

主要用户：在中国境内工作的外企/跨国企业职场人

| 维度 | 特征描述 |
| --- | --- |
| 英语水平 | 中级偏上（CET-6 / 雅思 6.5+），能读写但表达不够地道 |
| 工作场景 | 日常使用英文进行邮件、IM、会议、文档协作 |
| 痛点强度 | 高频依赖 AI 翻译/润色，但成果无法沉淀和复习 |
| 学习动机 | 实用主义，希望快速提升工作场景下的表达能力 |
| 时间特点 | 碎片化，难以系统学习通用英语课程 |

2.2 用户细分

1. 初入外企者（优先级：⭐⭐⭐）

   - 刚加入外企 0-6 个月

   - 需要快速掌握业务词汇和公司沟通风格

   - 对 AI 辅助工具接受度高

2. 外企老兵但表达瓶颈者（优先级：⭐⭐）

   - 工作 2-5 年，表达停留在“能用”层面

   - 希望突破到“专业、地道”表达

   - 有一定积累但缺乏系统复习

3. 跨部门协作频繁者（优先级：⭐⭐⭐）

   - PM、TPM、客户成功等角色

   - 每天需要与不同层级、不同文化背景的人沟通

   - 对场景适配和语气把控要求高

三、核心痛点分析

3.1 现状问题

| 痛点 | 具体表现 | 当前替代方案 | 方案缺陷 |
| --- | --- | --- | --- |
| 学习语料脱节 | 通用教材与实际工作场景差异大 | 购买“职场英语 100 句”类书籍 | 内容泛化，命中率低，学习效率差 |
| AI 成果无沉淀 | 每次 AI 润色后的优质表达无法复用 | 在 ChatGPT/Gemini 中反复提问 | 上下文混杂，无法结构化回顾 |
| 缺乏场景组织 | 工作沟通天然分场景，但记录散乱 | 手动在笔记软件中分类 | 成本高，难以坚持 |
| 没有内化闭环 | 看过、用过，但没记住 | 依赖记忆力 | 遗忘曲线陡峭，能力无法沉淀 |

3.2 用户旅程中的关键卡点

``\
现有流程：\
收到英文消息 → 理解大意 → 想回复但不确定表达 → 打开 AI 工具\
→ 输入背景和中文意思 → 获得英文回复 → 复制粘贴发送 → 【结束】\
↑\
断层点\
`

断层点：用户获得了一个好的表达，但这个表达没有被记录、分类、复习，下次遇到类似场景，还是要重新问 AI。

四、产品解决方案

4.1 产品核心逻辑

`┌─────────────────────────────────────────────────────────┐ │ 职场英语助手 │ │ │ │ 输入层 沉淀层 内化层 │ │ ┌─────┐ ┌─────┐ ┌─────┐ │ │ │润色 │ --> │结构化│ --> │复习 │ │ │ │助手 │ │语料库│ │系统 │ │ │ └─────┘ └─────┘ └─────┘ │ │ ↓ ↓ ↓ │ │ AI 驱动 场景标签 Flashcard │ │ 中英对照 词汇提取 间隔复习 │ │ 语境感知 时间轴 场景模拟 │ └─────────────────────────────────────────────────────────┘`

4.2 核心功能模块

模块一：智能润色助手

功能目标：替代散落在各处的 AI 对话，提供统一的沟通润色入口

核心能力：

- 中文 → 英文翻译 + 润色

- 英文 → 英文优化（提升地道度、专业度）

- 支持输入语境信息（对话背景、收件人角色、沟通目的）

- 中英文对照输出

- 提供多个表达方案供选择

输入字段：

- 原始内容（必填）

- 场景类型（邮件/Slack/会议/Jira/汇报）

- 语境说明（选填，如“回复客户投诉”）

- 语气偏好（正式/中性/友好）

模块二：结构化语料库

功能目标：自动归档每次润色记录，按场景、时间、词汇组织

核心能力：

- 自动场景标签（基于用户选择 + AI 识别）

- 关键词汇/短语自动提取

- 时间轴视图（按日期查看沟通记录）

- 场景视图（按邮件、Slack、会议等分类）

- 词汇视图（高频词汇统计和索引）

- 支持手动编辑和补充

数据结构：\
`沟通记录 Record { id: string created_at: timestamp scene_type: enum (email, slack, meeting, jira, report) context: string （用户输入的语境） original_text: string （原始中文/英文） polished_text: string （润色后英文） key_phrases: string[] （提取的关键表达） tags: string[] （自定义标签） review_count: number （复习次数） mastery_level: enum (new, learning, mastered) }`

模块三：智能复习系统

功能目标：帮助用户真正记住和掌握这些表达

核心能力：

- 自动生成 Flashcard（正面：中文场景，背面：英文表达）

- 间隔复习算法（基于遗忘曲线）

- 场景化模拟练习（给定场景，用户尝试表达，AI 评估）

- 个人高频词汇表（按使用频率排序）

- 学习进度可视化

Flashcard 生成逻辑：

- 从沟通记录中提取关键句子

- 识别可复用的表达模式

- 生成“场景 + 表达”的卡片对

5.1 功能优先级矩阵

| 功能 | 用户价值 | 技术复杂度 | MVP 包含 |
| --- | --- | --- | --- |

| 沟通记录自动保存 | ⭐⭐⭐ | ⭐ | ✅ |
| 场景标签（手动） | ⭐⭐⭐ | ⭐ | ✅ |
| 场景标签（AI 自动） | ⭐⭐ | ⭐⭐⭐ | ❌ |
| 关键词汇提取 | ⭐⭐⭐ | ⭐⭐ | ✅ |
| 时间轴视图 | ⭐⭐ | ⭐ | ✅ |
| 场景视图 | ⭐⭐⭐ | ⭐ | ✅ |
| Flashcard 生成 | ⭐⭐⭐ | ⭐⭐ | ✅ |
| 间隔复习算法 | ⭐⭐ | ⭐⭐ | ❌ |
| 场景模拟练习 | ⭐⭐ | ⭐⭐⭐ | ❌ |
| 学习进度统计 | ⭐ | ⭐ | ❌ |

5.2 MVP 范围定义

MVP 核心流程：

1. 用户输入需要润色的内容（中文或英文）

2. 选择场景类型，可选填语境说明

3. AI 返回润色后的英文 + 中英对照

4. 自动保存为沟通记录，提取关键词汇

5. 用户可在“我的语料库”中按场景/时间查看历史记录

6. 用户可手动将记录添加到 Flashcard 进行复习

MVP 不包含：

- AI 自动场景识别

- 智能间隔复习提醒

- 场景模拟对话练习

- 学习数据分析和报告

6.1 信息架构

v2.1 导航与模块拆分如下：

1. 顶层导航固定三块：
   - 语料库
   - 智能润色
   - 生词积累

4. 语料库（统一沉淀层）：
   - 智能润色语料（来自润色保存）
   - 生词摘录库（来自生词积累保存）
   - 统一时间线（同一时间轴合并展示两类新增记录）

七、用户故事（User Stories）

7.1 核心用户故事

Story 1: 快速润色 Slack 消息

- 作为 刚入职的外企员工

- 我想要 快速将我的中文回复润色成地道的英文

- 以便于 我能自信地在 Slack 上与同事沟通

验收标准：

- 用户可以在 30 秒内完成输入、润色、复制的完整流程

- 润色结果包含中英对照

- 自动提取 2-5 个关键表达

---

Story 2: 按场景查看历史沟通

- 作为 需要准备英文汇报的产品经理

- 我想要 查看我过去所有“汇报”场景下的沟通记录

- 以便于 我能快速找到可复用的表达和句式

验收标准：

- 用户可以按场景类型筛选历史记录

- 每条记录显示时间、原文、润色结果、关键词

- 支持关键词搜索

---

Story 3: 通过 Flashcard 复习表达

- 作为 想提升英语能力的外企员工

- 我想要 定期复习我积累的高频表达

- 以便于 我能逐渐内化这些表达，减少对 AI 的依赖

验收标准：

- 用户可以手动将沟通记录添加到 Flashcard

- Flashcard 正面显示中文场景，背面显示英文表达

- 用户可以标记“记得”或“不记得”

Story 4: 查看个人高频词汇

- 作为 想了解自己表达习惯的用户

- 我想要 看到我工作中最常用的英文词汇和短语

验收标准：

- 系统自动统计词汇使用频率

- 按频率降序展示

- 每个词汇显示使用次数、常见场景、例句

## 十三、与 Duolingo 的差异化分析

### 13.1 核心差异对比

| 维度 | Duolingo | WorkLingo （职场英语助手） |
| --- | --- | --- |
| **目标人群** | 语言零基础到中级学习者 | 已有中级英语能力的职场人 |
| **学习动机** | 兴趣驱动、考试驱动、旅游需求 | **工作刚需驱动**（不学就影响工作表现） |
| **学习内容** | 标准化课程（日常对话、旅游、商务） | **个性化语料**（用户自己的工作沟通） |
| **学习场景** | 碎片化、游戏化、主动学习 | **工作流嵌入**（边工作边学习） |
| **价值验证** | 完成课程、通过测试 | **工作中立即能用**（明天的会议就能说） |
| **激励机制** | 连续打卡、积分、排行榜 | **工作成果 + 能力提升**（见下文详述） |

### 13.2 目标人群的本质差异

**Duolingo 用户**：

- “我想学英语”（目标模糊）

- 学习是**额外任务**，需要外部激励（游戏化、社交压力）

- 学习成果难以即时验证（学了不知道有没有用）

**WorkLingo 用户**：

- “我明天要开英文会议，我必须能说”（目标明确）

- 学习是**工作的一部分**，内生动力强

- 学习成果立即验证（今天学的明天就能用上）

**关键洞察**：

> Duolingo 的用户需要“被说服去学习”，而 WorkLingo 的用户已经在学习了（他们每天都在用 AI 翻译、润色），我们只是把这个过程**结构化、可沉淀化**。

### 13.3 学习动机与激励机制设计

#### 问题：学习是违背人性的，激励是什么？

**Duolingo 的激励逻辑**：

```plaintext
外部激励（游戏化）→ 短期多巴胺 → 连续打卡 → 习惯养成
                                    ↓
                                 但容易断裂
                              （因为缺乏内在价值）
```

**WorkLingo 的激励逻辑**：

```plaintext
工作刚需 → 使用工具 → 立即见效 → 持续使用 → 能力提升
   ↓                      ↓                    ↓
内在动力              短期激励            长期激励
（不用会尴尬）      （今天就能用）      （升职加薪）
```

#### 三层激励体系设计

**第一层：即时激励（工作成果）**

这是 WorkLingo 最大的优势——**学习和工作是同一件事**。

| 场景 | 即时激励 |
| --- | --- |
| 润色邮件 | “邮件发出去了，客户回复了，沟通顺畅” |
| 准备会议 | “会上能流畅表达，获得认可” |
| 写 Jira 评论 | “同事理解了我的意思，问题解决了” |

**产品设计**：

- 每次润色后显示：“✅ 已优化，可直接使用”

- 鼓励用户标记“已使用”，并记录使用场景

- 统计“本周帮你完成了 23 次沟通”

**第二层：短期激励（能力进步可视化）**

不同于 Duolingo 的虚拟积分，WorkLingo 的进步是**真实的、可量化的**。

**产品设计**：

```plaintext
┌─────────────────────────────────────────┐
│  📊 你的进步                             │
│                                         │
│  本周新增高频表达: 12 个                 │
│  复习掌握: 8 个                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                         │
│  🎯 能力提升                             │
│  • 邮件回复速度提升 40%                  │
│    (从平均 15 分钟降到 9 分钟)           │
│  • 不再需要 AI 帮助的场景: 3 个          │
│    ✓ 日常时间安排                       │
│    ✓ 会议纪要请求                       │
│    ✓ 简单进度汇报                       │
│                                         │
│  💡 下一个目标                           │
│  掌握"客户投诉回复"场景的 5 个核心表达   │
│  (已掌握 2/5)                           │
└─────────────────────────────────────────┘
```

**关键指标**：

- **AI 依赖度下降**：从 100% 依赖 AI 到 50% 能自主表达

- **表达速度提升**：从思考 5 分钟到脱口而出

- **场景覆盖度**：从只会 3 种场景到能应对 15 种场景

**第三层：长期激励（职业发展）**

这是最强的内在动力，但需要产品帮助用户“看见”。

**产品设计**：

- **成就系统**（不是游戏化的徽章，而是真实的职场里程碑）

  - 🎤 “首次独立主持英文会议”

  - 📧 “连续 30 天无需 AI 辅助回复日常邮件”

  - 🌟 “向 VP 完成英文季度汇报”

- **能力证明**（可导出的学习报告）

  - “3 个月掌握 200+ 职场高频表达”

  - “完成 50+ 场景化口语练习”

  - 可用于简历、晋升申请

- **社区见证**（可选的社交功能）

  - 分享“我如何在 3 个月内从依赖翻译到独立开会”

  - 职场英语能力成长故事

### 13.4 为什么不需要“绿色小鸟”？

**Duolingo 需要游戏化的原因**：

- 学习内容枯燥（背单词、做语法题）

- 学习价值延迟验证（不知道什么时候能用上）

- 用户容易放弃（因为没有外部压力）

**WorkLingo 不需要过度游戏化的原因**：

1. **工作本身就是压力**：“明天要开会”比“小鸟催你”有效 100 倍

2. **价值即时验证**：今天学的明天就能用，不需要虚拟奖励

3. **目标用户是成年职场人**：他们要的是效率和结果，不是娱乐

**但我们需要的是**：

- ✅ **清晰的进度反馈**（我学到哪了？还差多少？）

- ✅ **成就感的仪式化**（完成里程碑时的庆祝）

- ✅ **温和的提醒**（不是“小鸟哭泣”，而是“你准备好明天的会议了吗？”）

### 13.5 产品定位的再明确

基于以上分析，WorkLingo 的定位应该是：

> **不是“英语学习 App”，而是“职场英语工作流工具 + 能力沉淀系统”**

**类比**：

- Duolingo = 健身房（你要专门去，需要自律）

- WorkLingo = 智能手环（嵌入日常，自动记录，顺便提升）

**Slogan 建议**：

- ❌ “让英语学习更有趣”（这是 Duolingo）

- ✅ “把每次 AI 帮你写的，变成你自己会说的”

- ✅ “工作中学英语，英语中做工作”

- ✅ “不是学英语，是用英语”

## 十四、总结

职场英语助手（WorkLingo）解决的是一个真实且普遍的痛点：**如何将 AI 辅助沟通的成果转化为个人能力**。它不是翻译工具，也不是通用英语学习平台，而是一个帮助外企职场人将日常工作中的真实英文沟通转化为可持续学习和复习的个人语料库的工具。

通过“输入 → 沉淀 → 内化”的闭环设计，我们希望用户能够逐步减少对 AI 的依赖，最终实现从“依赖 AI 润色”到“自主地道表达”的能力跃迁。

### 核心差异化优势

1. **工作流嵌入**：不是“额外的学习任务”，而是“工作的一部分”

2. **个性化语料**：学的都是自己工作中真实用到的表达

3. **即时价值验证**：今天学的明天就能用，不需要等待

4. **内生动力驱动**：工作刚需 > 游戏化激励

**下一步行动**：

1. 完成技术选型和架构设计

2. 开发 MVP 核心功能

3. 招募 20-30 名种子用户进行内测

4. 根据反馈快速迭代

5. 重点验证：用户是否真的会从“依赖 AI”过渡到“自主表达”
